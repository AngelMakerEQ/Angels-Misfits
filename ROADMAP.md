# Angels Misfits Development Roadmap

## Project Vision

Angels Misfits is a private EverQuest Emulator server developed for personal gameplay and long-term preservation of a classic EverQuest experience.

The goal is to create a stable, enjoyable, and maintainable personal server that captures the spirit of classic EverQuest while allowing carefully considered improvements that support solo development, administration, and long-term enjoyment.

The server is not intended for public release. Development priorities should focus on:

- Personal gameplay experience
- Historical authenticity where players notice it
- Long-term maintainability
- Reliable administration
- Enjoyable solo progression
- Learning and improving EQEmu development skills

Public-server concerns such as large-scale player management, community onboarding, donation systems, and external administration workflows are not project priorities unless the project direction changes.

---

# Development Philosophy

Angels Misfits should preserve the defining characteristics of classic EverQuest while avoiding unnecessary limitations caused by historical technology.

Prioritize authenticity in areas directly experienced while playing:

- World design
- Zone atmosphere
- NPC behavior
- Encounters
- Loot
- Itemization
- Progression
- Class identity
- Visual presentation

Allow modernization or customization in areas that improve personal enjoyment and maintainability:

- Administration tools
- Quality-of-life improvements
- Development workflows
- Debugging
- Solo gameplay support
- Server management

The goal is not to recreate every historical limitation. The goal is to recreate the feeling of playing classic EverQuest.

- Database baseline documentation
- Client configuration documentation
- Development workflow documentation

---

# Phase 2: Classic Experience Restoration

Goals:

Improve the modern RoF2 client experience to better represent classic EverQuest eras.

Potential areas:

- Classic zone assets
- Classic loading screens
- Classic UI elements
- Titanium-era UI investigation
- DuxasUI compatibility
- NPC model restoration
- Classic visual presentation

Player model philosophy:

- Allow optional Luclin player models
- Prefer classic-era NPC appearances
- Avoid Luclin-era NPC replacements unless intentionally chosen

---

# Phase 3: Database Baseline and Gameplay Foundation

Goals:

Understand and stabilize the starting database.

Tasks:

- Compare Angels Misfits database against PEQ
- Analyze TAKP database differences
- Document imported systems
- Identify unnecessary modern systems
- Establish database change tracking

Focus areas:

- Zones
- NPCs
- Loot
- Spawns
- Itemization
- Experience rates
- Progression systems

---

# Phase 4: Solo Gameplay Experience

Goals:

Create a satisfying personal EverQuest experience.

Potential systems:

- Solo-friendly progression adjustments
- Mercenary alternatives if desired
- Custom encounters
- Custom quests
- Quality-of-life improvements
- Personal progression systems

Changes should preserve the feeling of EverQuest rather than simply making the game easier.

---

# Phase 5: Content Development

Goals:

Expand and customize the world.

Potential areas:

- Quest scripting
- Custom NPCs
- Custom encounters
- Itemization adjustments
- Zone improvements
- New progression paths

All custom content should be documented before becoming permanent.

---

# Phase 6: Long-Term Maintenance

Goals:

Maintain a stable personal server.

Tasks:

- Document major changes
- Maintain database backups
- Track custom modifications
- Validate updates before applying
- Preserve compatibility with EQEmu development where practical
